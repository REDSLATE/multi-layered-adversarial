# Pulse Contract — REDEYE → Camaro audit row

> **Purpose.** Define the exact JSON shape REDEYE emits and RISEDUALAI's
> `_emit_camaro_audit` consumes. One source of truth, two repos.

This file lives in **REDEYE/runtime** (this repo). The consumer
(`_emit_camaro_audit`, `RedeyePulseCard`, `RedeyeCamaroFeedCard`) lives in
**RISEDUALAI/Camaro** (other repo). Forward-compat rule below keeps both
sides shippable independently.

---

## 1) Required fields (every payload)

| Field | Type | Notes |
|---|---|---|
| `engine` | `"REDEYE"` | Exact literal. Anything else → reject. |
| `symbol` | `str` | Uppercase. |
| `side` | `"SHORT"` | REDEYE only emits short-side advisories. |
| `action` | `"SHORT"` \| `"HOLD"` | `SHORT` only when bear_score ≥ 0.70 **and** confidence ≥ 0.62 **and** no liquidity/borrow block. |
| `bear_score` | `float` ∈ [0,1] | Rounded to 4 dp. |
| `confidence` | `float` ∈ [0,1] | Rounded to 4 dp. |
| `risk_multiplier` | `float` ∈ [0, 0.75] | Capped at `MAX_REDEYE_RISK_MULTIPLIER`. 0 when `action="HOLD"`. |
| `allowed` | `bool` | `True` iff Camaro is allowed to consider this short. Never means "execute". |
| `reason` | `str` | `+`-joined trigger reasons; `\|`-suffix carries blocked_reasons. |
| `reports_to` | `"CAMARO"` | Exact literal. |
| `created_at` | ISO 8601 UTC | `datetime.now(timezone.utc).isoformat()`. |
| `raw` | `dict` | `{features, model_score}` — replayable input. |
| `camaro_contract` | `dict` | See §2. |

## 2) `camaro_contract` block (immutable)

```json
{
  "source": "REDEYE",
  "role": "short_side_advisor",
  "may_execute": false,
  "may_override_alpha": false,
  "final_authority": "CAMARO"
}
```

**Camaro must reject** any payload where:
- `may_execute` is anything other than `false`
- `may_override_alpha` is anything other than `false`
- `final_authority` is anything other than `"CAMARO"`
- `source` is anything other than `"REDEYE"`

The block is generated by `export_for_camaro()` and is not user-tunable.

---

## 3) `alpha_alignment` (forward-compat field)

REDEYE's read on whether this short contradicts Alpha's current long thesis.

| Value | Meaning | Camaro's expected handling |
|---|---|---|
| `null` (default) | REDEYE has no view | Standard short-advisory flow. |
| `"aligned"` | Alpha is flat / bearish; REDEYE short reinforces | May size up modestly. |
| `"divergent"` | Alpha has no position on this name | Independent short; standard flow. |
| `"contradicts"` | **Alpha is currently long this name; REDEYE short directly contradicts** | **Escalate. Patent J / dual-sign territory if elevation is implied.** |

### Forward-compat rule

- **REDEYE always emits** `alpha_alignment` (defaults to `null`).
- **RISEDUALAI tolerates absence** — `_emit_camaro_audit` reads with
  `.get("alpha_alignment")`. If a older REDEYE build doesn't set it, the
  audit row stores `null`. No exception.
- **Validation is REDEYE-side only**. Bridge raises `ValueError` if a value
  outside `("aligned","divergent","contradicts")` is passed in.

This means RISEDUALAI can ship the audit-row column today with no migration
risk, and REDEYE can ship the field whenever its model gives it a real signal.

## 4) Audit row (RISEDUALAI persists)

`_emit_camaro_audit` writes one row per accepted REDEYE payload into
RISEDUALAI's audit collection. Minimum columns expected by `RedeyeCamaroFeedCard`:

```python
{
    "engine":           "REDEYE",                    # from payload
    "symbol":           "TSLA",                      # from payload
    "action":           "SHORT" | "HOLD",            # from payload
    "bear_score":       0.8915,                      # from payload
    "confidence":       0.9295,                      # from payload
    "risk_multiplier":  0.75,                        # from payload
    "alpha_alignment":  None | "aligned" | ...,      # from payload (default null)
    "reports_to":       "CAMARO",                    # from payload
    "reason":           "...",                       # from payload
    "camaro_contract":  {...},                       # from payload
    "received_at":      "<iso utc>",                 # RISEDUALAI server time
    "created_at":       "<iso utc>",                 # REDEYE wall time
    "may_execute":      false,                       # mirrored from contract
    "raw":              {features, model_score},     # for replay
}
```

`_id` is intentionally omitted — RISEDUALAI projects it out before sending
the row to the Pulse / Feed cards.

## 5) Worked example — `contradicts` case

REDEYE input:
```python
build_redeye_short_signal(
    symbol="TSLA",
    features={
        "price_change_pct": -2.4, "rsi_14": 39, "macd_hist": -0.22,
        "volume_ratio": 1.8, "below_sma_20": True, "below_sma_50": True,
        "failed_bounce": True,
    },
    model_score=0.82,
    alpha_alignment="contradicts",
)
```

Emitted to Camaro:
```json
{
  "engine": "REDEYE",
  "symbol": "TSLA",
  "side": "SHORT",
  "action": "SHORT",
  "bear_score": 0.8915,
  "confidence": 0.9295,
  "risk_multiplier": 0.75,
  "allowed": true,
  "reason": "negative_price_momentum+weak_rsi+...+redeye_model_score_blended",
  "reports_to": "CAMARO",
  "alpha_alignment": "contradicts",
  "created_at": "2026-02-09T20:20:21Z",
  "raw": { "features": {...}, "model_score": 0.82 },
  "camaro_contract": {
    "source": "REDEYE",
    "role": "short_side_advisor",
    "may_execute": false,
    "may_override_alpha": false,
    "final_authority": "CAMARO"
  }
}
```

Camaro's expected response: log audit row, **do not auto-act**. If a
position elevation is on the table, route through the existing dual-sign
primary countersign workflow (Build 3) so two distinct operators must sign
when REDEYE's contradicting short would force Alpha out of a long.

---

## 6) Versioning

This contract is `v1`. Add new optional fields freely; mark them
forward-compat (REDEYE always emits, RISEDUALAI tolerates absence). Never
remove or repurpose a field at the same name in v1 — bump to `v2` and add
a `contract_version` key to both sides first.
